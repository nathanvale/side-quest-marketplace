# PARA Philosophy Reference

Distilled from Tiago Forte's Building a Second Brain methodology.

## The Core Distinction: Projects vs Areas

| Aspect | Project | Area |
|--------|---------|------|
| **Definition** | Has a specific outcome AND deadline | Standard to maintain over time |
| **End state** | Can be "completed" | Ongoing, never "done" |
| **Timeline** | Short-term (days to months) | Long-term (continuous) |
| **Metric** | Success = outcome achieved | Success = standard maintained |
| **Verb** | "Complete", "finish", "deliver" | "Maintain", "manage", "handle" |

### Examples

**Projects:**
- "Launch new website" (outcome: website live, deadline: March)
- "Plan Melanie's birthday" (outcome: party happens, deadline: her birthday)
- "Migrate to PostgreSQL" (outcome: migration complete, deadline: Q2)

**Areas:**
- "Health" (ongoing responsibility, no completion)
- "Finances" (continuous management)
- "Home" (perpetual maintenance)

## The Relationship: Projects Fall Within Areas

Every project serves an area. They're interdependent:

```
Area: 🏡 Home
├── Project: 🎯 Install Solar Panels
├── Project: 🎯 Renovate Kitchen
└── Project: 🎯 Set Up Home Office
```

When the projects complete, the area continues. The area's standard ("maintain a comfortable, functional home") generates new projects over time.

## The Three Questions Pattern

When organizing any note, ask in order:

1. **What project does this support?**
   - If active project → link to project
   - If no active project → question 2

2. **What area is this relevant to?**
   - If ongoing responsibility → link to area
   - If neither → question 3

3. **Is this a resource to reference later?**
   - If useful reference → 03 Resources
   - If none of the above → 04 Archives (or delete)

## Organize by Actionability, Not Subject

**Wrong:** Organizing by topic (all health stuff together)
**Right:** Organizing by actionability (active project vs ongoing area vs reference)

The same topic can appear in multiple places:
- **Project:** "🎯 Run Marathon" (active goal)
- **Area:** "🏃 Health" (ongoing responsibility)
- **Resource:** "📚 Marathon Training Guide" (reference material)

## Common Classification Mistakes

### Mistake 1: Projects Without Deadlines

```
"Improve my coding skills"  ← Too vague, no end state
```

**Fix:** Make it specific:
- "Complete TypeScript course by March" ← Project
- OR keep as area: "Software Development" (ongoing skill maintenance)

### Mistake 2: Areas as Projects

```
"Manage my finances"  ← This is an area, not a project
```

**Fix:** Extract specific projects:
- Area: "💰 Finance"
- Project: "🎯 Set Up Budget System"
- Project: "🎯 Open Investment Account"

### Mistake 3: Orphaned Projects

Projects not connected to an area often drift or get abandoned.

**Fix:** Every project should answer: "What area of my life does this serve?"

## Project People vs Area People

Some people naturally think in projects (goal-oriented, deadline-driven). Others think in areas (maintenance-oriented, standard-driven).

**Project People:**
- Love checking things off
- Need deadlines to motivate
- Risk: neglecting ongoing maintenance

**Area People:**
- Great at sustaining effort
- Maintain high standards
- Risk: never completing anything specific

**The solution:** Use both. Areas generate projects; projects improve areas.
